Version: 1.0; Status: Final

  • Model
    • Model class BusinessResponseMessage (XML element BusinessResponseMessage) used a data type flavour that didn't allow a nullFlavor for model attribute interactionId (XML element interactionId). Changed data type flavour to II.NPfIT.oid.required. This relaxing of the data type is backwardly compatible.
    • Model class ManifestItem (XML element conveyingTransmission). Changed data type from II.NHS.IdentifierType2 to II.NHS.GenericType1 The previous data type was constrained to format of a DCE UUID. Payload identifiers can be in any format allowed by ID and IDREF. The new data type allows any string as a payload identifier. This change is backwardly compatible.
