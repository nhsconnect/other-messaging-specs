    • ITK wsdl
      • CoreServices-v1-0.wsdl and HSCI-v2-0.wsdl are deprecated and replaced by the ITKService.wsdl.
      • The rationale behind this change is to:
      • Reduce the overhead of supporting multiple wsdl files and multiple versions of wsdl files in future implementations when there is a message or interaction change.
      • Note 1:The use of the new ITKService.wsdl may require that current implementations need changes dependent on how web services have been implemented.
      • Note 2: Maintaining the current HSCI wsdl 2.0 would require major update for the new message names and versions, which would also require changes to existing implementations.
