1.  Introduction

This document details the conformance to the TermInfo recommendations.  It was created based on the January 2007 DSTU Ballot.


2.  Act.classCode

The values for this attribute shall be drawn from the HL7 ActClass vocabulary table as per TermInfo recommendations. Using SNOMED CT in HL7 Version 3; Implementation Guide, Release 1.2

3.  Act.code and Observation.value

We shall continue to make use of these attributes as has been done in the previous CFH messages, specifying the SNOMED CT code in the
Act.code attribute in all cases including where there is no associated value.

It is not intended that we will make use of the "ASSERTION" method which has been discussed in TermInfo. From section 2.2.2.2 point 5,
where Observation.value is omitted then the Observation shall be interpretted as equivalent to the same expression in Observation.value with
Act.code "ASSERTION"


Example:

Representation of 162053006 - supra-pubic pain
<observation...><code code="162053006" displayName="supra pubic pain".../></observation>

is the representation to be used.

<observation...><code code="ASSERTION".../><value code="162053006" displayName="supra pubic pain".../></observation>
should not be used.

4.  Act.moodCode

The values for this attribute shall be drawn from the HL7 ActMood vocabulary table as per TermInfo recommendations. Using SNOMED CT in
HL7 Version 3; Implementation Guide, Release 1.2

5.  Act.statusCode

The status code shall be taken from the constraint present on each of the models. Further details can be found in section 2.2.4 of the TermInfo
DSTU from the January 2007 ballot.

6.  Procedure.targetSiteCode and Observation.targetSiteCode

These attributes, defined by HL7 are not used. Instead the "procedure site" and "finding site" attributes from SNOMED CT shall be used as
qualifiers in a post-coordinated expression held in the CD datatype. Further details can be found in section 2.2.5 of the TermInfo DSTU from the
January 2007 ballot.

7.  Procedure.approachSiteCode and SubstanceAdministration.approachSiteCode

These attributes, defined by HL7 are not used. Instead the "approach" attribute from SNOMED CT shall be used as a qualifier in a post-coordinated
expression held in the CD datatype. Further details can be found in section 2.2.6 of the TermInfo DSTU from the January 2007 ballot.

8.  Procedure.methodCode and Observation.methodCode

These attributes, defined by HL7 are not used. Instead, the following attributes from SNOMED CT shall be used for each of the following types of expression:

  • procedure - method attribute

  • evaluation procedure - measurement method attribute

  • clinical finding - finding method attribute

Further details can be found in section 2.2.7 of the TermInfo DSTU from the January 2007 ballot.

9.  Act.priorityCode

The SNOMED CT attribute "priority" shall be used if the priority significantly alters the nature of the code. In other cases the HL7 attribute shall be used. Further
details can be found in section 2.2.8 of the TermInfo DSTU from the January 2007 ballot.

10.  Act.negationInd

This attribute, defined by HL7 shall not be used. Instead, the "finding context" and "procedure context" attributes from SNOMED CT shall be used as a qualifier in a post-coordinated expression held in the CD datatype. Further details can be found in section 2.2.9 of the TermInfo DSTU from the January 2007 ballot.

11.  Act.uncertaintyCode

This attribute, defined by HL7 shall not be used. Instead, the "finding context" attributes from SNOMED CT shall be used as a qualifier in a post-coordinated expression held in the CD datatype. Further details can be found in section 2.2.10 of the TermInfo DSTU from the January 2007 ballot.
