COCD_TP146010UK04
RiskToPerson

The highlighting of a risk to a Care Professional or third party which should be brought to the urgent attention of Care Practitioners.

Risk to Care professionals and Third parties are created by relating a coded entry (usually a Finding , Diagnosis or Social Context ) and relating to a risk to person act.

[1..1] [M] code (CD {CNE:RiskToPersonSnCT } )

A SnomedCT code taken from the vocabulary RiskToPersonSnCT.

The Concept Descriptor (CD) with CDA URL datatype/flavour shall be used to carry this information as follows:

  • The code attribute shall contain a code from the relevant vocabulary
  • The codeSystem attribute shall contain the OID for SnomedCT
  • The displayName attribute shall carry a description associated with the code in SnomedCT
  • The originalText element can be used to carry a reference element. This can link the coded item to the relevant text in the CDA section
  • qualifier (s) may be used if required

Note the implementation guidance for the domain where this template is used should be consulted regarding correct usage of the originalText element.

[1..1] [M] effectiveTime (IVL<TS>)

The time or duration that the risk to person was relevant.

The date range shall be carried as follows:

If a single point in time is to be carried the Date or Time Point data type shall be used:

  • The center attribute shall contain the date.
Used if the exact date/time is not known / available

If both a start date and an end date are present the Date or Time Interval Complete data type shall be used:

  • The low attribute shall contain the start date.
  • The high attribute shall contain the end date.

If only a start date is present the Date or Time Interval After data type shall be used:

  • The low attribute shall contain the start date.
This is used to indicate that the risk is still relevant

If only an end date is present the Date or Time Interval Before data type shall be used:

  • The high attribute shall contain the end date.

This is used to indicate that the risk is no longer relevant but the start dat/time is / was not known.

[0..1] author

Refers to the person who is the author of the risk to person.

The author is deemed to be the person who identified the risk unless the responsible party is used then that person is deemed to be the person who indentified the risk.

Author is only required if the author or author time is different from the inherited author.

[1..1] [M] contextControlCode (CS {CNE:ContextControl } ) { Fixed="OP" }

Specifies how this author participation contributes to the context of the risk to person observation and whether it may be propagated to descendent acts whose association allows such propagation.

"O" (Overriding)- the association replaces any context of the same or more specific type that has been conducted from the parent act.

"P" (Propagating)- the association is conducted to all child acts where the ActRelationship.contextConductionInd is set to "true".
