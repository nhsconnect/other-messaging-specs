Refers to the patient to whom any results returned relate.

A patient may have zero requests as the patient may never have been referred. A client system (e.g. a GP system or Healthspace) could legitimately use this interface to determine if a user has any active referrals. If the patient has no active referrals then the no requests would be returned, however this is not an error condition so an empty container object would be returned.

Used by: Appointment
[1..1] classCode (CS {CNE:AGNT} ){ Fixed="AGNT" }
Indicates that this role is of type agent.
  • Fixed value: "AGNT"
[1..1] id (II)
The SDS User ID of the agent.

The Identifier External datatype flavour is used to carry this information as follows:

  • The root attribute shall contain an OID with the value "1.2.826.0.1285.0.2.0.65"
  • The extension attribute shall contain the SDS User ID, known as the User's Unique Identifier (UUID) in the External Interface Specification.
