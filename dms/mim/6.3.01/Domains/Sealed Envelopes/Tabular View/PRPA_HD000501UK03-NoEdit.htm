      [1..1] [M] id (II)

      This is a unique identifier for the item being referenced

      When the action code carried in AccessorChange.code is "5" (An additional workgroup now has access to some sealed information e.g. receipt of a direct communication) the identifier should be in one of the formats below:

      In the case of the addition of an accessor on a CDA document the reference will be the ClinicalDocument.setId of the communicated document e.g. Outpatient report, Discharge report or Emergency Department report.

      • The Identifier Global datatype flavour is used to carry the identifier as follows:
        • The root attribute shall contain a DCE UUID.

        In the case of the addition of an accessor to one or more local objects for the purposes of internal referral (i.e. those which do not require a CDA style document communication), the reference will be the local identifier of the referral.

        • The IdentifierExternal datatype flavour is used to carry this information as follows:
          • The root attribute shall contain the local OID
          • The extension attribute shall contain the local identifier
        • Note the OID is not specificied with the MIM as it is local to the system that recorded the sealing or unsealing event

        When the action code carried in AccessorChange.code is "6" (The new GP workgroup now has access to some sealed information) the reference will be the ID of the EhrExtract.

        • The Identifier Global datatype flavour is used to carry the identifier as follows:
          • The root attribute shall contain a DCE UUID.

        When the action code carried in AccessorChange.code is "7" (A replacement workgroup now has access to some sealed information due to organizational change) a reference is not required.
