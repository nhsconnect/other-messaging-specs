QURX_RM000003UK01
Query

The entry point to the query, which allows prescriptions to be selected by specifying the requester's details and (optionally) prescription status parameters.

Used by:
[1..1] requesterParameters (RequesterParameters)

A list of parameters used for specifying by the requseter's details.

RequesterParameters

A named list of parameters (name/value pairs) representing the selection criteria for querying by a requester's details.

Used by: Query

Parameter to specify the time interval within which the prescription was last active.

[1..1] onlyCancellationPending (OnlyCancellationPending)

Parameter to specify only those prescriptions with cancellation pending. 

This parameter is reserved for future use.

[1..1] organization.id (Organization.id)

LastActivityTime

Parameter for specifiyng the time interval within which the last dispense of the prescriptions occurred.

Used by: RequesterParameters
[1..1] value (IVL<TS>)

Provides the value for the valued element structure (name-value pair) for the element specified in the query response.

This data shall be carried using one of the following datatype flavours:

1. the Date or Time Interval After where:

  • the low.value attribute shall contain the last activity after date with YYYYMMDDHHMMSS precision; or

2. the Date or Time Interval Before datatype flavour where:

  • the high.value attribute shall contain the last activity before date with YYYYMMDDHHMMSS precision; or

3. the Date or Time Interval Complete where:

  • the low.value attribute shall contain the last activity after date with YYYYMMDDHHMMSS precision; and
  • the high.value attribute shall contain the last activity before date with YYYYMMDDHHMMSS precision.
[1..1] semanticsText (ST)

Provides a unique identification (the name of the name-value pair) to an element within a specified query response structure.

OnlyCancellationPending

Parameter for specifying prescriptions with cancellation pending.

This parameter is reserved for future use, as currently the ETP System will operate with the default value of false.

Used by: RequesterParameters
[1..1] value (BL)

Boolean flag to include or exclude prescriptions with cancellation pending:

  • true if only prescriptions with cancellation pending shall be present in the query response, or
  • false if only prescriptions without cancellation pending shall be present in the query response

The value shall be set to false by default.

[1..1] semanticsText (ST)

Provides a unique identification (the name of the name-value pair) to an element within a specified query response structure.

Organization.id

Parameter for specifying prescriptions issued, downloaded or dispensed by the identified Prescriber or Dispenser organisation.

Used by: RequesterParameters
[1..1] value (II)

This is the prescriber site id (NACS code) if the requester organisation is a prescriber or the dispensing site id (NACS code) if the requester organisation is the downloading or the last dispensing site.

[1..1] semanticsText (ST)

RequesterOrganizationType

Used by: RequesterParameters
[1..1] value (CV {CNE:RxEventQueryOrganizationType} )

Code indicating the requester organisation type.

The Coded with Codesystem datatype flavour is used to carry this information as follows:

  • the code attribute shall contain a value from the list of codes in the RxEventQueryOrganizationType vocabulary;
  • the codeSystem attribute shall contain the OID value referenced in the RxEventQueryOrganizationType vocabulary.
