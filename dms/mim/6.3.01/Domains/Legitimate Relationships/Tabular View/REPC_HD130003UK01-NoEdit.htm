[1..1] code (CV {CNE:SDSRegistrationType} )

A code to provide the type of registration.

In this context, the type of registration is the addition of a user (User Role Profile) to a Workgroup.

The Coded with Code System datatype flavour is used to carry this information as follows:

  • The code attribute will contain value "01" (Add User To Workgroup) from the list of codes in the SDSRegistrationType vocabulary;
  • The codeSystem attribute will contain the OID referenced in the SDSRegistrationType vocabulary.

    [1..1] code (CV {CNE:SDSObservationType} )

    A code to provide the type of observation.

    In this context, the type of observation is the status of the originating User Role Profile, specifically whether or not they have administration rights for identified workgroups only.

    The Coded with Code System datatype flavour is used to carry this information as follows:

    • The code attribute will contain value "01" (Identified Workgroups Only) from the list of codes in the SDSObservationType vocabulary;
    • The codeSystem attribute will contain the OID referenced in the SDSObservationType vocabulary.
    [1..1] value (BL)

    If true, it means that the originating URP has administration rights for identified workgroups only (they only have RBAC Business Function B0090, and not B0100 or B0001). This indicates to the message recipient that it must make a check that the originating URP has administration rights on the workgroup to which the specified URP is to be added.
