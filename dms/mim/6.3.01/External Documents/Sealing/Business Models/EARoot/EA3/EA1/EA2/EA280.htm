Constraint Type Status Detail
Goal 1. Goal Approved
The Goal is for the Patient to be informed of changes - or access by unauthorized Users to their Sealed information.
Description 2. Description Approved
The Use case describes the process of Healthspace receiving an alert, processing it, and sending a message to the Patient.
Trigger 3. Trigger Approved
The Spine sends an alert to Healthspace.
Pre-Condition 4. Pre-condition Approved
Healthspace will be sent an alert for either of the following reasons.

An unauthorized user has accessed a Patients Sealed Information.

The Patients information has had it's sealing status changed.

A workgroup has being added to a Patients Sealing metadata.
Successful Post-Condition 5.Post-condition Approved
Patient has received notification
Guaranteed Post-Condition 5.Post-condition Approved
No alert is generated.
