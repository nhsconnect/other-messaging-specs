Update Care Record Demographic Data Constraint
Use Case Detail
1. Goal For a user to update (add, edit, delete) their demographic data in PDS
2. Trigger User decides to update their demographic data.
3. Pre-condition Record to be updated must exist - except in the case of adding a record.
4. Success Post-condition User successfully updates their demographic data.
5. Guaranteed Post-condition If the user is not able to update the data, a page is displayed describing the error.
