Update HealthSpace Summary Constraint
Use Case Detail
1. Goal The goal is to allow HealthSpace to maintain a complete and up to date collection of HealthSpace Summary data.
2. Trigger User initiates the task when they wish to add to, amend or remove some of their HealthSpace Summary data.
3. Pre-condition The data items that the user wishes to modify already exists
4. Success Post-condition HealthSpace Summary data has been successfully added, amended or removed.
5. Guaranteed Post-condition If the user is not able to modify the data, a page is displayed describing the error.
