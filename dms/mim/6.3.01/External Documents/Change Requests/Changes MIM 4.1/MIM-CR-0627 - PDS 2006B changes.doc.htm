Short Title of Issue

Changes and additions to the PDS domain for 2006B.

 

 

Description

A number of changes and additions are required to the PDS domain for the 2006B Spine Release, as follows:

  • Remove "Death Update Request", "General Update Request" and "Consent Update Request" messages.
  • Rename "Combined Update Request" to "General Update Request".
  • Add classes to carry Proxy consent to call-back in Successful Retrieval and Combined (General) Update messages.
  • Allow death date and death notification status to be removed using the Combined (General) Update message.
  • Add a mechanism for batching the Trace, Trace Response, Retrieval Request and Successful Retrieval messages. This will be a generic batching mechanism of existing PDS messages. The mechanism will be documented in the infrastructure section of the MIM, and messages which can be batched will reference this mechanism.
