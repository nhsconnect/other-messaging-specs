Short Title of Issue

 

Lack of documentation and implementation guidelines in areas of batching

 

 

Description

 

There needs to be additional guidance given on the following areas in batching:

 

i)     How do implementors know when they should be batching interactions. MIM currently says optional for PDS. What are the rules, is batching mandatory?

ii)    MIM needs to state that sender/recipient should not be used when the interaction is in a batch and the sender/recipient is carried in the batch wrapper.

iii)    MIM needs to give more information about error handling eg. Whole batch in error, some messages in error some not. Check that the EIS is updating error codes.
