issue Description:

Issue 15269

========

Due to an incorrect CMETInfo.txt declaration of NPFIT Agent CMETS, several instances of these CMETs in ETP RMIM diagrams carry the designation (ROL) rather than (AGNT).

 

Issue 16115

========

To support paperless prescribing, the information printed on the right-hand  side of the FP10 needs to be conveyed in the HL7 payload. For ETP R2, an interim solution of using "AdditionalInstructions" will be used. For ETP R3, a better solution is required.

 

Issue 17914

========

A new "Not Dispensed Reason" vocab is required for "Not Allowed on FP10" to cover prescribing of blacklisted and similar drugs that are not allowed on an FP10.

 

Issue 17917

========

Withdraw ETP Message (PORX_RM136001UK31) contains an 'Author' entity that uses the CMET "R_AgentNPFITPersonSDS". This appears to be an error as the CMET should be "R_AgentNPFITPerson" to align with the other messages in MM and because the tabular view states that various attribute within this CMET should be populated.

 

 

Proposed solution

Issue 15629

========

Suggest updating these elements as and when actual message changes are required.

 

Issue 16115

========

New entity off "Prescription"

Title "Supporting Information"

Cardinality 0..1

Containing just a simple plain text attribute (I'll let you work out what this would be in HL7)

Description = "Supporting information related to the prescription. For example, a list of the patient's authorised repeat medication."

 

Issue 17914

========

Update code set as requested

 

Issue 17917

========

This is believed to be a tabular view error.  The W5 does not specify the required author information, so retain CMET but remov constraints from tabular view description as all elements are mandatory.
