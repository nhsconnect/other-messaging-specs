Short Title of Issue

No mechanism interrogating the ETP database if a dispensing system fails with loss of data.

Description

 

Bugzilla issue

 

15751

 
If a dispensing system fails resulting in data loss there is not currently a mechanism that allows them to interrogate the ETP database to determine the prescriptions that they have responsibility for and their status.
 
It is therefore proposed to put in place functionality that will enable dispensing systems to retrieve the latest state of prescriptions as recorded by the ETP component of the Spine.  This will allow them to partially rebuild their local data sufficiently to allow continued dispensing and claiming for medication dispensed.
 
I order to support this new functionality a set of Prescription Query Messages is required.

Perceived Impact

New set of prescription query artefacts required.

Reason for issue being raised

Enhancement request by ETP team as part of changes for ETP v3 / Spine release 2007-A.
