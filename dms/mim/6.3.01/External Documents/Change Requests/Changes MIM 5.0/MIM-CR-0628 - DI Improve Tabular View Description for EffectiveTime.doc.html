Short Title of Issue

Changes to Diagnostic Imaging IM file

Description

Bugzilla Issue 14014 from BT NASP.

 

Description: “EffectiveTime is widely misused as the time at which the information was entered into the system. This is not correct - that is availability time.

 

Effective time of an observation is the time at which the observation was made (i.e. the time the observation was effective).”

 

Perceived Impact

Mostly Tabular View Changes to EffectiveTime descriptions.

Plus For Diagnostic Imaging Request - POII_RM030000UK01

JustifyingInformationText – requires a Visio change {IVL<TS>}, Schemas and Examples need regenerating

EstimatedDeliveryDate – requires a Visio change (change from required to mandatory)

 

Backward Compatibility Status

 

Reason for issue being raised

Bugzilla Issue 14014 from BT NASP.

 

Suggested Action

Mostly changes the right hand side Tabular View descriptions for EffectTime as follows :

 

Diagnostic Imaging Request - POII_RM030000UK01

 

LastMenstrualPeriod

The date of the start of the patient’s last known Menstrual Period

 

JustifyingInformationText

The date and time that the relevant clinical information text is entered, i.e. the time of what the statement is about. This is not supporting text of another observation; it’s the justifying the observation itself (although it does support the request). IVL<TS> is used to describe effective time in JustifyingInformation (coded), so it should follow that it is used in this instance too.  NB A change to the Visio is required too. Schemas and Examples need regenerating

 

PregnancyStatus

The date and time that the status is applied to identify whether the patient is definitely, is definitely not, or is possibly pregnant.

 

EstimatedDeliveryDate

The estimated date of delivery of the children that the patient is carrying.  This is required but not mandatory, which is surely not right when you are stating a delivery date.  Therefore, this needs to be made to be mandatory.  NB A change to the Visio is needed.

 

NotPregnantReason

Identifies the date and time that the reason was determined as to why the patient can safely say they are not pregnant. E.g. if the reason had a time (eg hysterectomy), then it should be entered here.

 

 

Diagnostic Imaging Report - POII_RM040000UK30

 

CodedDiagnosis

The date and time when the Reporting HCP identifies the diagnosis determined by this procedure, i.e. the relevant time that the diagnosed condition has.

 

 

DIProcedureSummary

The Date & Time the DI Procedure was performed, checked for quality and then completed (extracted from the W5).

 

 

CTT Resolution and Current Status

Proposed Change:  Changes to the Tabular Views, some cases require changes to the RMIM Visios and also schemas and examples, as outlined.
