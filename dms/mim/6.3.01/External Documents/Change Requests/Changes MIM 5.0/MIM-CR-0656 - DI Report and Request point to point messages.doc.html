Short Title of Issue

 Point to Point Messages.

Description

Requirements for Point to Point messages (DI):

 

These will only be point to point and will NOT go to PSIS. As such, they:-

 

  • do not need to comply with the Care Provision Pattern
  • do not need to provide a PSIS Focal act or Event
  • do not need to carry CRE Types
  • will not have Snapshot IDs (They will be ‘required‘ as 0..2)
  • Do not need Presentation Text
  • Must have an II in the Focal Act to facilitate whole message replacement.

 

In both types, Act References may only be used within the actual message instance. If information is required that would normally be referenced, it may be repeated (with a different Snapshot id for Care Provision Pattern messages sent to PSIS). If Snapshot IDs are not required for Point to Point, this would immediately prevent Act references.

 

New artefact versioning system, resulting in all artefacts being re-versioned to the following new format:

 

POII_XX000nnnUKnn:
