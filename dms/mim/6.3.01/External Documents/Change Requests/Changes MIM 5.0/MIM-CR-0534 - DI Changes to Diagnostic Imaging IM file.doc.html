Short Title of Issue

Changes to Diagnostic Imaging IM file

Description

The following errors occur in the Diagnostic Imaging IM file:

  1. Overview – states that DI Reports are single modality only – this is incorrect as Reports may be multi-modality
  2. Overview – states that “It is assumed that a DI Procedure will be complete prior to a report on it being issued”  - this should read “It is assumed that a DI Procedure will be completed or aborted prior to a report on it being issued”
  3. Application Role: DI Fulfiller – text states “By default, the Primary Recipient of the report is the Placer of the original request.” This is incorrect – the primary recipient of the report is the HCP with clinical responsibility for the patient.
  4. Application Role: DI Report Receiver – text states “…the identified Health Care Professional (HCP) who …” – this abbreviation has already been stated.
  5. Trigger Event: DI Report Release – text states “The report may be 'Complete' or still 'Active'.” – this is incorrect according to the HL7 vocab ActStatus and should read ‘completed’ or ‘active’. Further mentions of ‘Complete’ and ‘Active’ are made elsewhere in the textual description also.
  6. Interaction: DI Report Withdrawal – trigger event is stated as ‘DI Report Release’ – this is incorrect and should be ‘DI Report Withdraw’
  7. Report Status section – text states “When a Diagnostic Imaging (DI) Report message” – the ‘DI’ abbreviation has already been stated
  8. Report Status section – text states “Each Request for DI Procedures is created on the Fulfiller system” – this should read “Each Fulfiller Request for…”.
  9. Report Status section – throughout reference is made to the status of Procedure Report Segments, Summary Report Segments, and Placer Requests – none of these have a status
  10. Report Status section – reference is made to the ‘Procedure’ act – this should be ‘DIProcedure’ act
  11. Report Status section – Scenario3, second table: description for ‘Replacement of’ states “Previous report Focal act ID (The previous Report becomes Obsolete)” – this is confusing re the status of the payload of the message or the actual report
  12. Report Status section tables – reports the following available states for:

·         DI Report – active, completed, obsolete, or aborted – DI Report may only be active or completed

·         Procedure - active, completed, obsolete, or aborted – DI Procedure may only be completed or aborted

·         Fulfiller Request Details - active, completed, obsolete, or aborted – Fulfiller Request Details may only be active or completed

 

 

Perceived Impact

Minor – changes to IM file only

 

Backward Compatibility Status

 

Reason for issue being raised

Errors in IM file

 

Suggested Action

The following changes should be made:

  1. Overview – change text to state that DI Reports may be multi-modality
  2. Overview – change text from “It is assumed that a DI Procedure will be complete prior to a report on it being issued” to “It is assumed that a DI Procedure will be completed or aborted prior to a report on it being issued”
  3. Application Role: DI Fulfiller – change text to “The Primary Recipient of the Report is the system of the Healthcare Professional with clinical responsibility for the patient.”
  4. Application Role: DI Report Receiver – change text to “…the identified HCP who …”
  5. Trigger Event: DI Report Release – replace mentions of ‘Complete’ to ‘completed’ and ‘Active’ to ‘active’.
  6. Interaction: DI Report Withdrawal – change trigger event to ‘DI Report Withdraw’
  7. Report Status section – change “When a Diagnostic Imaging (DI) Report message” to “When a DI Report message”
  8. Report Status section – change text from “Each Request for DI Procedures is created on the Fulfiller system” to “Each Fulfiller Request for DI Procedures is created on the Fulfiller system”
  9. Report Status section – remove reference to the status of Procedure Report Segments, Summary Report Segments, and Placer Requests
  10. Report Status section – replace reference to ‘Procedure’ act with ‘DIProcedure’
  11. Report Status section – Scenario3, second table: change description for ‘Replacement of’ to “Previous report Focal act ID shall be set to obsolete”
  12. Report Status section tables – remove reference to the following states:

·         DI Report – obsolete, aborted

·         Procedure - obsolete, aborted

·         Fulfiller Request Details - obsolete, aborted
