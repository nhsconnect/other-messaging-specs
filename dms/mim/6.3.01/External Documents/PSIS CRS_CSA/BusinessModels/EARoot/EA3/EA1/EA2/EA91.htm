Appears In: Send Report Detailed Use Case
This is the process that will allow a user to send a report.

The report will contain information regarding the patient's care episode that needs to be stored on PSIS.

The report could be a new report or a replacement of an existing report already on PSIS.

Trigger Process Approved
The process begins when a user decides to create, edit or verify a report.
User logged in Pre-condition Approved
The user has to be logged into the system for the process to begin. The report can be created at any time during or after the care episode. Typically the report will only be completed and sent when the care episode has finished.
Report saved (draft) Post-condition Approved
Report has been saved in draft form.
Report created but needs verification Post-condition Approved
The Report has been created but could be verified at the time. The Report will need to be re-loaded and verified by an authorised user.
Report received Post-condition Approved
The recipient system has received the report with no errors.
Report received (PSIS) Post-condition Approved
PSIS has received the report with no errors.
Report(s) received with errors Post-condition Approved
The Report(s) have been sent and an error response has been received by the Local System.
