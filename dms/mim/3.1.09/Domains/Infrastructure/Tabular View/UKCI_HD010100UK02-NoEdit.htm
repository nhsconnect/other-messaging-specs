WithdrawMessage

A message used to notify a recipient that a message the recipient has already received should not have been sent.

Used by:
[1..1] classCode (CS {CNE:ALRT} ){ Fixed="ALRT " }

Indicates that the message is acting as an alert.

  •  Fixed value: "ALRT"
[1..1] moodCode (CS {CNE:EVN} ){ Fixed="EVN " }

Indicates that this is an alert that has happened.

  • Fixed value: "EVN"
[1..1] id (II)

The unique identifier of the message that is being withdrawn.

This is the Focal Act or Event Id identifier from the Focal Act of the specific domain payload message being withdrawn. 

The Identifier Global datatype flavour is used to carry this information as follows:

  • A DCE UUID is held in the root attribute
[1..1] recordTarget
Refers to the patient who was the record target of the message being withdrawn.
[1..1] typeCode (CS {CNE:RCT} ){ Fixed="RCT " }

Indicates the role of patient as record target.

  • Fixed value: "RCT"
[1..1] patient (Patient)

The person who is the patient. If a withdrawal message is being sent then this must be included.

[1..1] id (II)

To provide the nationally recognised identifier for the patient, the NHS number.

The Identifier External datatype flavour is used to carry this information as follows:-

An NHS number is carried in the following way:

  • The root attribute will contain an OID with the value "2.16.840.1.113883.2.1.4.1"
  • The extension attribute will contain the NHS number.
