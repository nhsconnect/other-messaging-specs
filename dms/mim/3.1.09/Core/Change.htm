2.2

Core Technical Team

15/04/2004

Patch to update eBS

 

eBS Updates:

Change requests : DACM-CR-24, DACM-CR-26 & DACM-CR-27.

 

  • Replaced all references to GMP code with SDS User Id in tabular views
  • Removed mandated use of SNOMED CT  from Referral tabular view
  • Re-versioned all artifacts that have changed since V2.0

 PRPA_RM030000UK03

  • The following have been changed from PRPA_RM030000UK02

Substance

  • formCode vocabulary amended from MaterialForm to ClinicalProductForm
  • code vocabulary changed from SubstanceSnct to Substance

ObservationStatement

  • code vocabulary  changed from ObservationCodeSnct to ObservationCode

ProcedureEvent

  • code vocabulary changed from ProdecureCodeSnct to ProcedureCode

 

Associated Vocabulary Updates:

eBooking ReasonForReferralRejection vocabulary added

 

Use of Snomed-CT codes is no longer mandatory in any eBooking ‘code’ attributes, i.e. in ObservationSnct, ProcedureSnct, SubstanceSnct, ClinicalProductForm and EhrCompoundCode.  As a result the ‘Snct’ element has been removed from the names of the first three.  Descriptions of all five - and EhrLevelCode - amended, and also have hyperlinks in their descriptions to (new) vocabularies describing the coding systems that can be used (see below). 

new vocabularies added for:

- the Read Codes 4-byte set

- the Read Codes Version 2

- Clinical Terms (Read Codes) V3

- Snomed-CT (UK subset)

- Egton Codes

- EMIS Drug Codes.

 

Vocabulary index updated accordingly.

 

This version references V2.1 of the eBooking Implementation Manual, V2.1 of the Infrastructure Management Implementation Manual, V1.0 of the Legitimate Relationships Implementation Manual, V1.1 of the PDS Implementation Manual, V2.2 of the Vocabulary and V2.1 of Data types
