QueryByParameter
A heading for providing information about the query, in this case a request to Choose and Book for outline information about a patient's referrals.
Used by:

A heading for providing information about the NHS number of the patient making the request. This value is only provided if the enquirer is a patient.
[0..1] enquirerRoleProfileCode (EnquirerRoleProfileCode)
A heading for providing information about the Enquirer Role Profile Code the Role Profile Code of the user making the request, if the user is a professional user.
[1..1] patientNHSNumber (PatientNHSNumber)
A heading for providing information about the Patient NHS number, A number used to identify a person uniquely within the NHS in England and Wales.
This parameter value is mandatory and so must contain a value.
EnquirerNHSNumber
A heading for providing information about the NHS number of the patient making the request. This value is only provided if the enquirer is a patient.
Used by: QueryByParameter
[1..1] value (II)
To provide the value of the parameter item (identified in the associated semanticsText element) to be searched.
An NHS Number for the enquirer must always be provided when the enquirer is a patient.

If provided, normally this will be the same as the patient's NHS number but in the case of a proxy acting on behalf of a patient, it will be different.

As with PatientNHSNumber, the Identifier External datatype flavour is used to carry this information as follows:-

A NHS number is carried in the following way:

  • The root attribute will contain an OID with the value "2.16.840.1.113883.2.1.4.1";

  • The extension attribute will contain the NHS number itself.

EnquirerRoleProfileCode
A heading for providing information about the Enquirer Role Profile Code the Role Profile Code of the user making the request, if the user is a professional user.
Used by: QueryByParameter
[1..1] value (II)
To provide the value of the parameter item (identified in the associated semanticsText element) to be searched.

The Role Profile Code of the user making the request, is required to check if the requester is authorised to request information about referrals.

The Role Profile Code is an SDS identifier that uniquely identifies the role profile of the person.   This is carried in the following way:

  • The root attribute shall contain an OID with the value "1.2.826.0.1285.0.2.0.67"
  • the extension attribute shall contain the SDS Role Profile ID, known in the External Interface Specification as Role Profile Code.
