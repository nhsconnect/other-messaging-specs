An uninitialised request is created when Choose and Book receives a Request Service message from a GP system. The system should then perform a seamless transition into Choose and Book to allow the user to intialise the request and create an appointment request or advice and guidance request.

However, if for some reason the user does not transition into Choose and Book, or the referral status message is requested before they have initialised the request, an uninitialisedRequest will be returned.
