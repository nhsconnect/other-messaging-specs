The search criteria are used to create a shortlist to search the Choose and Book Directory of Services.

Currently a user must specify one or more of:
- clinic type
- searchKeyword
- named clinician

Specialty may be used in conjunction with searchKeyword and/or named clinician, and must be used in conjunction with clinic type.

any of the following may also be included in the search criteria:
- searchDistance
- searchOrganisation
- maximumIndicativeWaitTime

The following boolean values will always be supplied, and are defaulted to false:
- extendedChoiceIndicator
- ignoreAgeRestriction
- ignoreGenderRestriction.

The search criteria are clinical information.

public int
  maximumIndicativeWaitTime		 Range:0 to 1
Notes: This is used to further filter the priority, if the priority is ‘2 week wait’ then the indicative wait time (IWT) is ignored. If the priority is urgent or routine then it is used to filter out services with no urgent or routine slots (as appropriate) within the specified IWT.
public int
  namedClinician		 Range:0 to 1
Notes: SDS ID of the named clinician that was searched on.
public clinicalTerm
  searchKeyword		 Range:0 to 1
Notes: It is possible to search for services which have been defined as supporting a particular clinical term (whether this is a condition treated or procedure performed). The search can be further refined by using the searchKeyword in conjunction with other search criteria.

The searchKeyword will always be a coded value with the textual representation returned for information.
