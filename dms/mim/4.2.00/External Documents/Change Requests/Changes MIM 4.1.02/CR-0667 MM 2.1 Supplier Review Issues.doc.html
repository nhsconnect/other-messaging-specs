Short Title of Issue

MM 2.1 Supplier Review Issues.

Description

 

Bugzilla issue

 

15251

 
There is no indication of how to calculate the 18-character short-version prescription id, nor any reference to the compliance documentation where the algorithm is given. 
At least put a reference in to the ETP Prescribing Systems Compliance specification (similar to the reference that is included for digital signatures)

15255

 
The ETP example Parent Prescription does not give a valid 18-character short code (the checksum is incorrect even if ABCDEF is a valid NACS code).
Provide a valid example. Assuming the ETP service validates the check digit, suppliers will need to unit test their code or their mesages will be rejected in testing.

15411

 
The description of responsibleParty is also represented in the Patient CMET.  Accenture believes this is the better option, but this needs to be registered GP or registered practice as per PDS.

15412

 
The description of responsibleParty is not clear. Patients are now registered with a practice, not a GP.  Is this the registered GP?  Why is it mandatory? Does this need to be populated when the responsibleParty is the same as the author?

15453

 
MIM-CR-0516 - MM Vocabulary changes to support backwards compability Issue CR not addressed.  The following vocabularies require additional values to support backwards compatibility changes for ETP 1.6.  PrescriptionReleaseRejectionReason (existing Vocab) NNNN - Rejected due to Version problem  DispenseReturnRejectionReason(existing Vocab) NNNN - Rejected due to Version problem

Minimal impact with no version changes required  change to tabular views and vocabularies only.
