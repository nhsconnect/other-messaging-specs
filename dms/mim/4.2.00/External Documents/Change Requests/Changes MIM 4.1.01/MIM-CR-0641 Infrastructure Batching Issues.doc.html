Short Title of Issue

Each Manifest Item should take the Identifier External datatype and needs an OID.

 

Description

 

All identifiers carried in ManifestItems need to be in the same format,

i.e. Identifier  External or Identifier Global, but not a mixture of both.

Identifer External is the only option, in order to differentiate different

types of identifier. An OID needs to be defined for message id.

 

Perceived Impact

 

No impact

 

Backward Compatibility Status

No issues - correction

Reason for issue being raised

 

Issue raised by supplier – BT NASP.

 

Suggested Action

 

Add OID to message id item in manifest item list

 

CTT Resolution and Current Status

 

This does not conform to HL7 and I am reluctant to do it but there is no easy workaround so I have bowed to pressure from BT and allocated an OID as they requested.

 

The unique message id is carried as a UUID currently but need to be assigned an OID – affects tabular views only.
