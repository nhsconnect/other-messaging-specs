    GPConnect-Task-Order-1-0

    The order resource describes the task that is being communicated.

    Follow this link to view examples for GPConnect-Task-Order-1-0: Examples
    NameCard.TypeDescription & Constraints
    OrderOrderThe order resource supports the task being communicated

    taskDescription0..*extension-gpconnect-task-description-1-0Task Description

    identifier1..1IdentifierA unique identifier for the task

    system0..1UriThe namespace for the identifier
    Example Value: http://www.acme.com/identifiers/patient or urn:ietf:rfc:3986 if the Identifier.value itself is a full uri

    value1..1StringThe value that is unique
    Example Value: 123456

    period0..1PeriodTime period when id is/was valid for use

    start0..1DateTimeStarting time with inclusive boundary

    end0..1DateTimeEnd time with inclusive boundary, if not ongoing

    date0..1DateTimeWhen the order was made

    subject0..1gpconnect-patient-1-0Patient this task is about

    source0..1gpconnect-organization-1-0, gpconnect-practitioner-1-0Who initiated the order

    target0..1gpconnect-organization-1-0Who is intended to fulfill the order

    reason[x] (Type)1..1CodeableConceptReason for the task
    Binding: Indicates the reason for the task that is being communicated. (http://fhir.nhs.net/ValueSet/gpconnect-reason-type-1-0)
    Binding Strength: Required

    detail1..1gpconnect-encounter-1-0The encounter associated with the task

    reference0..1StringRelative URL reference

    display0..1StringText alternative for the resource
