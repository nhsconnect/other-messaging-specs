Version 1.0 : DRAFT B

    GPSoC-Schedule-1-0

    The Schedule resource provides a container for time-slots that can be booked using an appointment.

    Follow this link to view examples for Schedule: Examples
    NameCard.TypeDescription & Constraints
    Schedule1..1DomainResourceThis schedule resource is constrained to support communication of appointment and scheduling details with in scope of the GPSoC IM2 project.

    identifier (Schedule Identifier)0..*IdentifierAn identifier used to identify the schedule.

    system0..1uriThe namespace for the identifier
    Fixed Value: http://fhir.nhs.net/Id/gpsoc-schedule-identifier

    value0..1stringThe value that is unique
    Example Value: SCH001

    type0..*CodeableConceptType of appointment schedule
    Binding: Additional details about where the content was created (e.g. clinical specialty) (http://hl7.org/fhir/ValueSet/c80-practice-codes)
    Binding Strength: Required

    coding0..1CodingA code taken from the clinical specialty list of codes.

    system1..1uriIdentity of the terminology system
    Fixed Value: http://hl7.org/fhir/ValueSet/c80-practice-codes

    code1..1codeSymbol in syntax defined by the system
    Example Value: 394803006

    display1..1stringRepresentation defined by the system

    actor1..1StructureDefinition/gpsoc-location-1-0The resource this Schedule resource is providing availability information for. This can be a schedule for a Location or a Practitioner.

    planningHorizon0..1PeriodThis is the period of the appointment schedule

    start0..1dateTimeStarting time with inclusive boundary

    end0..1dateTimeEnd time with inclusive boundary, if not ongoing

    comment0..1stringComments on the availability to describe any extended information. Such as custom constraints on the slot(s) that may be associated
