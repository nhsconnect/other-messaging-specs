    ADW-AssessmentNotice-Message-Header-1-0

    ADW-AssessmentNotice-Message-Header-1-0

    MessageHeader
    Definition

    The reference(s) that are the subject of the action as well as other information related to the action are transmitted in a bundle in which the MessageHeader resource instance is the first resource in the bundle.
    Control 0..*
    Type MessageHeader
    Must Support True
    Comments
    MessageHeader.timestamp(ADW Message Sent Time)
    Definition

    The date and time on which the Assessment Notice was issued.
    Control 1..1
    Type Instant
    Must Support True
    Requirements

    Allows limited detection of out-of-order and delayed transmission. Also supports audit.
    Comments
    MessageHeader.event(Assessment Notice)
    Definition

    A fixed code to describe the type of ADW message.
    Control 1..1
    Binding The type of ADW message
    The codes SHALL be taken from http://fhir.nhs.net/ValueSet/adw-message-event-1-0
    Type Coding
    Is Modifier True
    Must Support True
    Requirements

    Drives the behavior associated with this message.
    Comments
    MessageHeader.event.system(Assessment Notice system)
    Definition

    The identification of the code system that defines the meaning of the symbol in the code.
    Control 1..1
    Type Uri
    Requirements

    Need to be unambiguous about the source of the definition of the symbol.
    Comments

    The identification of the code system that defines the meaning of the symbol in the code.
    Fixed Value http://fhir.nhs.net/ValueSet/adw-message-event-1-0
    MessageHeader.event.code(Assessment Notice Type)
    Definition

    A symbol in syntax defined by the system. The symbol may be a predefined code or an expression in a syntax defined by the coding system (e.g. post-coordination).
    Control 1..1
    Type Code
    Requirements

    Need to refer to a particular code in the system.
    Comments
    Fixed Value 975701000000106

    Must Support True

    MessageHeader.destination(Local Authority Receving System)
    Definition

    The destination application which the message is intended for.
    Control 1..1
    Type BackboneElement
    Must Support True
    Requirements

    Indicates where message is to be sent for routing purposes. Allows verification of "am I the intended recipient".
    Comments

    There SHOULD be at least one destination, but in some circumstances, the source system is unaware of any particular destination system.
